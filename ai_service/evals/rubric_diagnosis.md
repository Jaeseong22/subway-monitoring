# 진단 에이전트 (diagnosis.py) 평가 루브릭

## 무엇을 재는가

진단 에이전트가 "잘 동작한다" = **로그에 실제로 존재하는 근본 원인을, 도구를 실제로
써서, 정확히 찾아낸다.** 원인이 불명확한 케이스에서는 억지 결론을 내지 않고 '미결'로 둔다.

에이전트는 ReAct 루프로 도구(breakdown_by / time_histogram / sample_events /
field_values)를 골라 로그를 파고든 뒤 근본 원인을 낸다.

## 정답을 정한 원칙 (구현 독립)

정답은 **현재 코드가 무엇을 내는지 보지 않고** 로그의 실제 패턴으로 정했다:

- **원인 명확(`resolution: 완료`)**: 오류/포화가 특정 엔드포인트·코드·시각에 집중되어
  단일 근본 원인이 로그에서 드러나는 경우. `cause_elements`에 원인에 반드시 포함돼야 할
  핵심 요소(엔드포인트, 오류 코드, 시간 집중 등)를 나열했다.
- **원인 불명확(`resolution: 미결`)**: 오류가 여러 코드·엔드포인트에 분산되어 패턴이
  없거나, 로그 표본이 극히 적어 신뢰성 있는 특정이 불가능한 경우. 이때 억지 결론은 오답이고
  '미결'이 정답이다.

## 평가 축

### A. 근본원인 정확성 (0-2) — **LLM-judge 자동 채점**
`root_cause`가 케이스의 실제 원인(`cause_summary`)과 일치하는가. 텍스트의 의미 일치라
규칙 기반 자동 채점이 불가능하므로 **별도 LLM(judge)** 에게 정답과 대조시켜 채점한다.
judge는 진단 에이전트와 독립된 호출이다.

- **2**: 실제 원인을 정확히 특정 (핵심 요소를 옳게 짚음)
- **1**: 방향은 맞으나 부분적/모호 (예: "API 오류"까지만, 어느 엔드포인트인지 놓침)
- **0**: 틀린 원인, 또는 로그에 없는 사실을 지어냄

미결 케이스는 A를 적용하지 않는다(원인이 없는 게 정답이므로 축 C로만 본다).

judge도 LLM이라 흔들릴 수 있어, judge 판정 근거를 함께 덤프해 사람이 검토할 수 있게 한다.

### B. 도구 활용 (0/1) — 자동 채점
에이전트가 도구를 실제로 써서 결론에 도달했는가. `steps_used > 0`이고
`investigation`이 비어 있지 않으면 1. 도구 없이 곧장 결론을 지어내면 0.

### C. 결론 적절성 (0/1) — 자동 채점
`status`가 `expected.resolution`과 일치하면 1.
- 원인 명확 케이스에서 '완료'면 1, '미결/생략'이면 0 (있는 원인을 못 찾음).
- 원인 불명확 케이스에서 '미결'이면 1, '완료'면 0 (없는 원인을 지어냄 — 위험).

### D. 조사 효율 (0-2) — 자동 채점
결론까지 쓴 도구 단계 수가 적정한가.
- **2**: 1~4 단계로 결론 (효율적)
- **1**: 5 단계 (max_steps=6 직전, 다소 비효율)
- **0**: 0 단계(도구 없이 결론) 또는 max_steps 소진 후 미결(파고들지 못함)

## 채점 실행

```bash
cd ai_service
set -a; source ../.env; set +a       # OPENAI_API_KEY 필요 (ES 불필요)
python evals/run_diagnosis.py                # A(judge), B, C, D
python evals/run_diagnosis.py --dump         # + 진단 원문/조사과정/judge 근거
```

케이스를 바꾸려면 `gen_diagnosis_cases.py`를 수정하고 재생성한다.

## 케이스 구성

| 범주 | 개수 | 정답 |
|---|---|---|
| resolved (원인 명확) | 6 | 완료 + 정확한 원인 |
| unresolved (원인 불명확) | 2 | 미결 (억지 결론 금지) |

실제 프로덕션 이상 데이터가 부족해, 케이스는 실제 detection/agent가 만드는 로그 형태
(event_type, http_status, error_code, endpoint, @timestamp)를 따라 합성했다.
`gen_diagnosis_cases.py`가 생성하고 `cases_diagnosis.jsonl`이 정본이다.
